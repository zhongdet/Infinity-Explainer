Infinity-Explainer 系統架構

遞歸文本解釋器 — 三層模塊設計與兩條互動路徑

三種名詞來源 (Layer 1)

  • • 策展清單 (curated) — 內建 dictionary.ts，不可刪除
  • • 用戶選取 (user) — 手動選取文字後標記，綁定 shapeId
  • • LLM 發現 (llm) — 解釋回傳時自動註冊，全域性
  • • TermRegistry 統一疊加，最長匹配優先

兩條互動路徑 (Layer 2)

  • • 路徑① 點擊 — EventBoundary 判斷 pointerDown 在 term span
  • → onTermClick → LLMService.explain() → 新窗口
  • • 路徑② 選取 — pointerUp 時 window.getSelection() 非空
  • → SelectionPopover 確認 → registerUserTerm() + LLM
  • • 路徑③ 無操作 — 放行事件回 tldraw

遞歸循環 (Layer 3)

  • • TextWindow → Tokenizer 掃描 → 發現名詞
  • • 點擊名詞 → LLM 生成解釋 + 子名詞
  • • 子名詞註冊 → 新 TextWindow 自動識別
  • • 循環持續，直到無新名詞可解釋
